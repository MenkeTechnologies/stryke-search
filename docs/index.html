What it is

stryke-search is an Elasticsearch / OpenSearch client for stryke. Both engines speak the same REST API, so one client covers both. It ships as a Rust cdylib that stryke dlopens in-process on the first use Search; a ureq HTTP keep-alive connection pool is cached per (base_url, auth) for the life of the process. No tokio, no OpenSSL — pure-Rust transport over rustls.

Install

s add github.com/MenkeTechnologies/stryke-search

Then use Search in any stryke script.

Quick start

use Search

var %conn
$conn{url} = "http://127.0.0.1:9200"

Search::index_create("books", body => { mappings => { properties => { title => { type => "text" } } } }, %conn)
Search::doc_index("books", { title => "rust in action" }, id => "1", %conn)
Search::index_refresh(index => "books", %conn)

val $hits = Search::search("books", Search::match("title", "rust"), %conn)
p $hits->{hits}{total}{value}
